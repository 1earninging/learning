# C++ 与 Python 内存模型专题笔记

## 第一部分：C++ 核心（底层控制与效率）

### 1. 数据传递的三种姿势
*   **值 (Value)**：深拷贝。修改副本，原件不变。`void func(int x)`
*   **指针 (Pointer)**：存储地址。需解引用 `*` 访问，可指向空 `nullptr`。`void func(int* x)`
*   **引用 (Reference)**：对象的别名。必须初始化，一旦绑定不可更改。**现代 C++ 优先使用引用**。`void func(int& x)`

### 1.5 Day1 必交付：安全的 `swap(T&, T&)`（含自交换）+ 为什么要用 move
*   **面试结论**：`swap` 之所以建议用 `move`，是为了对“可移动但拷贝昂贵”的类型（如 `std::vector`、`std::string`、自定义大对象）避免深拷贝，把“资源所有权”从 A 转到临时变量，再转到 B。
*   **自交换保护**：`swap(x, x)` 在很多类型上也能工作，但写成通用模板时，加上 `if (&a == &b) return;` 是一个很好的防御式习惯（尤其当你后续写了更复杂的逻辑/自定义类型）。
*   **推荐**：优先用标准库 `std::swap`（内部会尽可能走 move），手写是为了展示你理解 move/异常安全。

```cpp
// 参考可运行 demo：2026_0227/language/swap_safe.cpp
template <class T>
void my_swap(T& a, T& b) {
    if (&a == &b) return;            // 自交换：直接返回
    T tmp = std::move(a);            // 资源从 a “搬走”
    a = std::move(b);                // b 的资源搬到 a
    b = std::move(tmp);              // tmp 的资源搬到 b
}
```

### 2. Const 正确性 (Const Correctness)
*   **保护数据**：`const int& x` 既保证了传参的高效（不拷贝），又保证了函数内部不会意外修改数据。
*   **Const 成员函数**：类中不修改成员变量的函数必须加 `const` 后缀，否则 `const` 对象无法调用它。
    ```cpp
    int getValue() const { return value; } // 正确做法
    ```
*   **指针的 const 组合（高频追问）**：
    *   `const T* p`：**指向 const 的指针**（不能通过 `p` 改 `*p`，但 `p` 可以指向别处）
    *   `T* const p`：**const 指针**（`p` 不能改指向，但可以改 `*p`）
    *   `const T* const p`：两者都 const

### 3. 对象生命周期与内存分区
*   **栈 (Stack)**：自动管理，速度快。局部变量在离开作用域 `{}` 时自动销毁。
    *   *陷阱*：绝对不能返回局部变量的引用或指针。
*   **堆 (Heap)**：手动管理 (`new`/`delete`)。生命周期由程序员控制。
    *   *现代方案*：使用 **RAII**（如 `std::unique_ptr`）利用栈对象的析构函数来自动释放堆内存。
*   **返回值优化 (RVO) & 移动语义**：现代 C++ 返回 `vector` 等容器时，会通过“所有权转移”而非“内存拷贝”来提升性能。
*   **更常见的悬垂来源：容器扩容导致引用/指针/迭代器失效**
    *   例如 `std::vector` 发生扩容时会搬家到新内存，之前拿到的 `&v[0]`、`int& ref = v[0]`、迭代器都可能失效（再用就是 UB）。

## 第二部分：Python 核心（对象模型与标签）

### 1. 变量即“标签” (Everything is an Object)
*   Python 变量不是内存盒子，而是贴在对象上的**标签**。
*   `a = b` 的本质是让 `a` 和 `b` 指向同一个内存对象，增加引用计数。

### 2. 可变 vs 不可变 (Mutable vs Immutable)
*   **不可变 (int, str, tuple)**：修改操作实际上是创建了新对象，并重新绑定标签。
*   **可变 (list, dict, set)**：可以在原位 (In-place) 修改内容，所有指向该对象的标签都会看到变化。
    *   *奇葩案例*：元组不可变，但如果元组内包含列表，列表的内容是可以改的（元组只保证其持有的指针地址不变）。

### 3. 内存管理机制
*   **引用计数**：每个对象记录被引用的次数，归零时立即释放。
*   **循环引用与 GC**：Python 的垃圾回收器会定期清理互相引用但无法从外部访问的“孤岛”对象。
*   **一句话把参数传递讲清楚（面试标准表述）**：Python 是 **pass-by-sharing**（“传对象引用的拷贝”）。所以函数里对参数的“影响外部与否”，取决于：
    *   你是在**原地修改**可变对象（外部能看到），还是把形参**重新绑定**到新对象（外部看不到）。

## 第三部分：经典问题回顾（避坑指南）

### Q1: C++ 中 `int x = v[0]` 和 `int& y = v[0]` 的区别？
*   `x` 是拷贝，`v` 清空后 `x` 依然安全。
*   `y` 是引用（地址），`v` 清空后 `y` 变成**悬空引用**，访问会导致崩溃。

### Q2: Python 中 `a = [1,2]`, `b = a`, `a = [3,4]` 后 `b` 是多少？
*   `b` 是 `[1,2]`。因为 `a = [3,4]` 是将标签 `a` 重新绑定到了新对象，而 `b` 依然指向旧对象。

### Q3: Python 函数默认参数 `def func(lst=[])` 为什么危险？
*   默认参数在函数定义时只创建一次。多次调用如果不传参，会共用同一个列表对象，导致数据污染。

### Q4: `==` 和 `is` 的区别？
*   `==`：判断“长得像不像”（内容一致性）。
*   `is`：判断“是不是同一个对象”（内存地址一致性）。

### Q5: Python 浅拷贝 vs 深拷贝，怎么一句话区分？
*   **浅拷贝**：只复制最外层容器；内部元素（引用）仍然共享。
*   **深拷贝**：递归复制所有层级；修改内部对象不会互相影响。

```python
# 参考可运行 demo：2026_0227/language/copy_demo.py
import copy
a = [[1], [2]]
b = copy.copy(a)        # 浅拷贝
c = copy.deepcopy(a)    # 深拷贝
a[0].append(99)
print(b)  # 浅拷贝会看到 [[1, 99], [2]]
print(c)  # 深拷贝仍是 [[1], [2]]
```

## 第四部分：双语言对比总结

| 特性 | C++ | Python |
| :--- | :--- | :--- |
| **赋值语义** | 默认拷贝 (Copy) | 默认引用 (Reference/Label) |
| **内存释放** | 确定性（析构函数立即执行） | 非确定性（靠 GC 机制） |
| **参数传递** | 值/指针/引用可选 | 统一传递对象引用 |
| **优化手段** | 移动语义、RVO、内联 | 对象池（小整数缓存）、内置函数优化 |
