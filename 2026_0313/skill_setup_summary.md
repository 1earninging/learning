# Skill: 对话自动归档机制 (Auto-Archiver)

## 核心问题
如何在 Cursor 中建立一种“技能”或“工作流”，使得每次对话结束后，能够自动将对话的核心精华内容沉淀到以日期命名的文件夹中，方便日后查阅和复习。

## 解决方案
利用 Cursor 的 `.cursorrules` 文件定义系统级指令（System Prompt Rule），指导 AI 在特定触发条件下执行归档操作。

### 实现步骤
1.  **修改规则文件**：在项目根目录的 `.cursorrules` 中添加 `## 对话自动归档机制 (Skill: Auto-Archiver)` 章节。
2.  **定义触发条件**：
    - 对话自然结束时。
    - 用户明确指令：“保存”、“存档”、“总结本次对话”。
3.  **定义执行逻辑**：
    - 获取当前日期 (YYYY_MMDD)。
    - 检查/创建对应文件夹 (e.g., `2026_0313`)。
    - 生成 Markdown 文件，包含核心知识点、代码片段和后续计划。

## 关键代码/配置
在 `.cursorrules` 中添加的规则如下：

```markdown
## 对话自动归档机制 (Skill: Auto-Archiver)
为了沉淀学习成果，每次对话结束或当用户要求“存档/保存”时，请执行以下操作：

1.  **创建/确认目录**：
    - 获取当前日期（格式：YYYY_MMDD，如 2026_0313）。
    - 检查当前目录下是否存在该日期的文件夹，若不存在则创建。

2.  **生成核心精华文档**：
    - 在该日期文件夹下创建 Markdown 文件 (.md)。
    - 文件名应准确反映对话的核心主题（如 vllm_scheduler_analysis.md）。
    - 内容应包含：
        - **核心问题**：我们讨论解决了什么问题？
        - **关键知识点**：架构原理、源码逻辑或技术细节。
        - **代码/图表**：有价值的代码片段、Mermaid 流程图或 Excalidraw 链接。
        - **TODOs**：未解决的问题或下一步计划。
```
